FCC Amateur Callsign Database (hamdb)
=======================================

NAME
    hamdb - Download, maintain, and query a local MariaDB copy of the
    FCC amateur radio license database.

SYNOPSIS
    hamdb <command> [argument]
    hamdb <callsign>

DESCRIPTION
    hamdb is a third-party utility (https://github.com/k3ng/hamdb,
    Copyright (C) 2001-2024 Anthony Good), adapted in this repository
    for MariaDB 10.5+ (mariadb-import replaces the removed
    mysqlimport command). Its behavior is otherwise unmodified.

    DigiHub creates the "hamdb" MariaDB user, populates the database
    with "hamdb full" during installation, and refreshes it daily via
    the dhhamdbupdate systemd timer (see the README's "HamDB (FCC
    Callsign Database)" section). Most users only need "hamdb
    <callsign>" (or, more commonly, "qrz <callsign>", which layers
    HamDB API enrichment and geocoding on top).

    With no recognized command, hamdb treats the argument as a
    callsign and looks it up directly.

COMMANDS
    <callsign>
        Look up a callsign directly (equivalent to the default case
        below).

    full, populatedb, populatedatabase, updatedatabase, updatedb, update
        Download and import FCC data. "full" always forces a
        complete re-import. The other aliases import a full copy
        only if run on a Monday (i.e. the day after Sunday) or if
        the local "en" table is currently empty; otherwise they pull
        the smaller daily update file.

    getfccfile
        Download the full FCC data file only, without importing it.

    makedatabase, makedb
        Create the fcc_amateur database and its tables without
        importing any data.

    removedb
        Drop the entire fcc_amateur database.

    removetables
        Drop the en, am, hd, hs tables and their _temp variants.

    cleanup
        Remove temporary download/import files left in ~/hamdb.temp.

    zipcode <ZIP>
        List active amateurs whose ZIP code starts with <ZIP>.

    state <ST>
        List active amateurs licensed in state <ST>.

    like <CALLSIGN>
        Search active amateurs by callsign prefix (SQL wildcards
        allowed).

    last <LASTNAME>
        Search by last name.

    lastname <LASTNAME>
        Search active amateurs by last name.

    former <CALLSIGN>
        Look up a callsign's record and any former-callsign history.

    history <CALLSIGN>
        Show a callsign's licensing action history.

    lastissued
        List the most recently issued licenses (piped through
        "more").

    lastissuedstate <ST>
        Most recently issued licenses in state <ST>.

    lastissuedzip <ZIP>
        Most recently issued licenses whose ZIP starts with <ZIP>
        (partial ZIPs are fine).

    dumptableen
        Dump the full "en" (entity) table.

    dbcount, count, counts
        Show row counts for each table, and a status breakdown of
        the hd table.

    sql "<SQL STATEMENT>"
        Run an arbitrary SQL statement against the fcc_amateur
        database.

    -h, --help, help
        Display hamdb's built-in (abbreviated) command summary.

    -v, --version, version
        Display hamdb's version string.

DESCRIPTION (CONFIGURATION)
    On first run, if ~/.hamdb.cnf does not exist, hamdb prompts
    interactively for a MySQL/MariaDB username and password and
    saves them to ~/.hamdb.cnf (mode 700). DigiHub creates this file
    automatically during installation, so this prompt normally never
    appears; qrz reads the same file for its own database queries.

EXAMPLES
    Look up a callsign:
        hamdb W0FFS

    Force a full database re-import:
        hamdb full

    Run the same day-aware update the daily timer uses:
        hamdb update

    List active amateurs in ZIP 33558:
        hamdb zipcode 33558

    Run a custom query:
        hamdb sql "select count(*) from en"

FILES
    ~/.hamdb.cnf
        MySQL/MariaDB credentials (MYSQLUSERNAME, MYSQLPASSWD),
        mode 700. Created automatically by install.sh.

    ~/hamdb.temp
        Working directory for downloaded/imported FCC data files;
        cleaned up automatically after a successful import.

EXIT STATUS
    hamdb does not use a formal, documented set of exit codes; it
    generally reflects the exit status of the underlying wget/curl,
    unzip, and mysql/mariadb commands it runs.
